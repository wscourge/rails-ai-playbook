# Internationalization (i18n)

> No hardcoded English anywhere. Every user-facing string uses i18n — both Rails backend and React frontend.

---

## Overview

| Layer | Library | Usage |
|-------|---------|-------|
| **Ruby/Rails** | Built-in `I18n` | `I18n.t("key")` or `t("key")` in views |
| **React** | `react-i18next` | `useTranslation()` hook → `t("key")` |

Both layers share the same key conventions but maintain separate translation files.

---

## Backend (Rails I18n)

### Directory Structure

```
config/locales/
├── en.yml                # Default — shared/global keys
├── models/
│   └── en.yml            # ActiveRecord attribute names
├── interactors/
│   └── en.yml            # Interactor error messages
├── mailers/
│   └── en.yml            # Email subjects and content
└── controllers/
    └── en.yml            # Flash messages, API responses
```

### Key Conventions

```yaml
# config/locales/en.yml
en:
  common:
    save: "Save"
    cancel: "Cancel"
    delete: "Delete"
    confirm: "Are you sure?"
    loading: "Loading..."
    sending: "Sending..."

  errors:
    not_found: "Resource not found"
    not_authorized: "You are not authorized to perform this action"
    email_required: "Email is required"
    email_invalid: "Email format is invalid"
    email_taken: "Email is already taken"
    name_required: "Name is required"
    password_too_short: "Password must be at least 8 characters"

  auth:
    login_success: "Welcome back!"
    signup_success: "Account created successfully"
    logout_success: "Signed out"
    invalid_credentials: "Invalid email or password"
```

```yaml
# config/locales/controllers/en.yml
en:
  users:
    create:
      success: "User created successfully"
    update:
      success: "Profile updated"
    destroy:
      success: "Account deleted"

  teams:
    invitations:
      sent: "Invitation sent to %{email}"
      accepted: "You joined %{team_name}"
```

```yaml
# config/locales/interactors/en.yml
en:
  interactors:
    users:
      validate_signup_params:
        email_required: "Email is required"
        email_invalid: "Please enter a valid email address"
        email_taken: "An account with this email already exists"
        password_too_short: "Password must be at least 8 characters"
        name_required: "Name is required"
```

### Usage in Ruby

```ruby
# In interactors
context.fail!(error: I18n.t("errors.email_required"))

# In controllers
redirect_to root_path, notice: I18n.t("auth.signup_success")

# With interpolation
I18n.t("teams.invitations.sent", email: "user@example.com")

# In mailers
def welcome(user)
  mail(to: user.email, subject: I18n.t("mailers.welcome.subject"))
end
```

### Application Config

```ruby
# config/application.rb
config.i18n.default_locale = :en
config.i18n.fallbacks = true
config.i18n.load_path += Dir[Rails.root.join("config/locales/**/*.yml")]
```

## Frontend (react-i18next)

### Installation

```bash
bun add i18next react-i18next
bun add -D @modyfi/vite-plugin-yaml
```

### Vite Config

Add the YAML plugin so `.yml` files can be imported as JS objects:

```ts
// vite.config.ts
import yaml from "@modyfi/vite-plugin-yaml";

export default defineConfig({
  plugins: [
    yaml(),
    // ... other plugins
  ],
});
```

### Setup

```ts
// app/frontend/lib/i18n.ts
import i18n from "i18next";
import { initReactI18next } from "react-i18next";

// Import all locale files — each feature gets its own small YAML file
import common from "@/locales/en/common.yml";
import auth from "@/locales/en/auth.yml";
import validation from "@/locales/en/validation.yml";
import errors from "@/locales/en/errors.yml";
import nav from "@/locales/en/nav.yml";
// Add new feature files here as they're created

const en = {
  ...common,
  ...auth,
  ...validation,
  ...errors,
  ...nav,
};

i18n.use(initReactI18next).init({
  resources: { en: { translation: en } },
  lng: "en",
  fallbackLng: "en",
  interpolation: {
    escapeValue: false, // React already escapes
  },
});

export default i18n;
```

### Translation Files

**Keep translation files small — one per feature/domain.** They are merged at init time in `i18n.ts`. Each file is a YAML file with a top-level namespace key matching the filename — same format as Rails backend locale files.

```yaml
# app/frontend/locales/en/common.yml
common:
  save: Save
  cancel: Cancel
  delete: Delete
  confirm: Are you sure?
  loading: Loading...
  sending: Sending...
  search: Search...
  no_results: No results found
  previous: Previous
  next: Next
  close: Close
  copy: Copy
  copied: Copied!
```

```yaml
# app/frontend/locales/en/auth.yml
auth:
  email: Email
  password: Password
  name: Name
  sign_in: Sign In
  sign_up: Sign Up
  sign_out: Sign Out
  forgot_password: Forgot password?
```

```yaml
# app/frontend/locales/en/validation.yml
validation:
  email_required: Email is required
  email_invalid: Please enter a valid email address
  name_required: Name is required
  password_too_short: Password must be at least 8 characters
  message_too_short: Message must be at least 10 characters
```

```yaml
# app/frontend/locales/en/errors.yml
errors:
  something_went_wrong: Something went wrong
  try_refreshing: Please try refreshing the page
```

```yaml
# app/frontend/locales/en/nav.yml
nav:
  home: Home
  dashboard: Dashboard
  settings: Settings
  pricing: Pricing
```

When adding a new feature, create a new file (e.g., `locales/en/billing.yml`, `locales/en/teams.yml`) and import it in `i18n.ts`. Never let a single file grow beyond ~30–40 keys.

### Import in Entrypoint

```jsx
// app/frontend/entrypoints/inertia.jsx
import "@/lib/i18n";  // Initialize i18n before app renders
```

### Usage in Components

```jsx
import { useTranslation } from "react-i18next";

function LoginForm() {
  const { t } = useTranslation();

  return (
    <form>
      <label>{t("auth.email")}</label>
      <Input placeholder={t("auth.email")} />

      <label>{t("auth.password")}</label>
      <Input type="password" placeholder={t("auth.password")} />

      <Button type="submit">{t("auth.sign_in")}</Button>

      <Link href={routes.signup}>{t("auth.sign_up")}</Link>
    </form>
  );
}
```

### With Interpolation

```yaml
# app/frontend/locales/en/teams.yml
teams:
  invitation_sent: Invitation sent to {{email}}
  member_count: "{{count}} member"
  member_count_plural: "{{count}} members"
```

```jsx
t("teams.invitation_sent", { email: "user@example.com" })
// → "Invitation sent to user@example.com"
```

### In Validation Schemas (Zod)

```ts
// Validators use i18n too — import the instance directly
import i18n from "@/lib/i18n";
import { z } from "zod";

export const loginSchema = z.object({
  email: z.string()
    .min(1, () => i18n.t("validation.email_required"))
    .email(() => i18n.t("validation.email_invalid")),
  password: z.string()
    .min(8, () => i18n.t("validation.password_too_short")),
});
```

## Conventions

1. **Backend and frontend have separate translation files.** Don't try to share one file. They serve different purposes.
2. **Keep files small.** One file per feature/domain, ~30–40 keys max. Split when a file grows beyond that. Both Rails (`config/locales/` subdirectories) and React (`locales/en/` per-feature YAML) follow this pattern.
3. **Key structure mirrors file structure.** `auth.sign_in` for auth pages, `teams.invitation_sent` for team features.
4. **Use interpolation for dynamic values** — `%{name}` in Rails, `{{name}}` in i18next. Never concatenate strings.
5. **Keep keys short but descriptive.** `errors.email_required` not `errors.the_email_field_is_required`.
6. **No hardcoded user-facing strings in code.** Button labels, error messages, flash notices, form labels, placeholders — everything uses i18n.
7. **Server sends localized strings when possible.** For Inertia props, send `I18n.t("...")` values from the server rather than sending keys for the frontend to translate.
8. **Flash messages use i18n.** `redirect_to path, notice: I18n.t("controller.action.success")`.

## Checklist for New Features

- [ ] All Ruby strings use `I18n.t()`
- [ ] All React strings use `t()` from `useTranslation()`
- [ ] Validation messages (Zod) use `i18n.t()`
- [ ] Flash messages use `I18n.t()`
- [ ] Email subjects and content use `I18n.t()`
- [ ] Translation keys added to `config/locales/*.yml` and `app/frontend/locales/en/<feature>.yml`
- [ ] New feature file imported and merged in `i18n.ts` (if new file created)
